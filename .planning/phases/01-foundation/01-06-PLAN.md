<task type="auto">
  <name>Task 1: Create build-dependency-registry.js Script</name>
  <files>scripts/build-dependency-registry.js</files>
  <action>
    Create a Node.js script that builds the dependency registry:

    1. **Input**: Run `scripts/trace-dependencies.js` to get code-level dependencies (require/import)
       - trace-dependencies.js outputs JSON to stdout with file-level dependency edges
       - Parse its output, convert to JSONL records

    2. **Record format** per DEPENDENCY_STANDARD:
       ```json
       {
         "id": "dep-{uuid}",
         "timestamp": "ISO-8601",
         "version": "1.0.0",
         "source": "path/to/source.js",
         "target": "path/to/target.js",
         "type": "requires",
         "confidence": "high",
         "metadata": { "discoveredBy": "auto", "method": "import-scan" }
       }
       ```

    3. **Auto-discovered code deps**: confidence "high", type "requires"
       - Code dependencies from trace-dependencies.js are auto-accepted (no review needed)

    4. **System-level edges**: Add known system relationships manually in the script:
       - hooks -> hook libs (requires)
       - skills -> agents (invokes)
       - pre-commit -> ESLint plugin (validates)
       - CI workflows -> scripts (invokes)
       - hooks -> settings.json (references)
       - CANON standards -> enforcement scripts (validates)
       These are hardcoded in the script as "curated" entries with confidence "high"

    5. **Output**:
       - Write JSONL to `CANON/DEPENDENCY_GRAPH.jsonl` using atomic write (tmp+rename via safe-fs.js or manual implementation)
       - Generate `CANON/DEPENDENCY_GRAPH.md` markdown view with:
         - Summary table: total edges, edges by type, edges by confidence
         - Full edge list grouped by source system (hooks, skills, agents, scripts, etc.)

    6. **Script conventions**:
       - Use `path.join()` and `path.resolve()` for all paths (cross-platform)
       - Use `crypto.randomUUID()` for IDs
       - Handle errors with try/catch, meaningful exit codes
       - Add `#!/usr/bin/env node` shebang
       - Follow existing script patterns in scripts/ directory

  </action>
  <verify>
    - `node scripts/build-dependency-registry.js` runs without errors
    - `CANON/DEPENDENCY_GRAPH.jsonl` is created with valid JSONL (each line is valid JSON)
    - `CANON/DEPENDENCY_GRAPH.md` is created with summary and edge tables
    - `node -e "require('fs').readFileSync('CANON/DEPENDENCY_GRAPH.jsonl','utf8').trim().split('\\n').forEach(l=>JSON.parse(l))"` parses all lines
  </verify>
  <done>Script runs, discovers code dependencies via trace-dependencies.js, adds curated system edges, outputs JSONL + MD</done>
