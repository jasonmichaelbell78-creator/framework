  <action>
    Run verification checks for each requirement:

    **FOUND-01: Framework has a single unified config file with local override pattern**
    - Verify `framework.config.json` exists and is valid JSON
    - Verify it has `project` and `systems` sections
    - Verify `project.name` is set
    - Verify `framework.config.local.json` is in .gitignore
    - Verify `framework.config.local.json.example` exists showing override pattern
    - Attempt runtime validation: load the Zod schema and parse the config
      (`npx tsx -e "import {frameworkConfigSchema} from './CANON/schemas/framework-config.schema.ts'; ..."`)

    **FOUND-02: CANON directory contains schemas, standards, and templates**
    - Count schemas: `ls CANON/schemas/*.ts | wc -l` should be 2
    - Count standards: `ls CANON/standards/*.md | wc -l` should be 9
    - Count templates: `ls CANON/templates/*.md | wc -l` should be 8
    - Verify CANON_INDEX.md exists and has correct counts

    **FOUND-03: 10 CANON standards documents cover all system conventions**
    - List all 9 standard files (note: CONTEXT.md says "10 standards, 2 schemas, 8 templates" but per research there are 9 standards + AUDIT_TEMPLATE which is a template, not a standard -- reconcile this count)
    - Verify each has the 6 required sections: Purpose, Scope, Requirements, Enforcement, Escape Hatches, Version History
    - Verify each uses RFC 2119 keywords

    **FOUND-04: Dependency registry populated with known relationships**
    - Verify `CANON/DEPENDENCY_GRAPH.jsonl` exists with valid JSONL
    - Verify records have all required fields (id, timestamp, version, source, target, type, confidence)
    - Verify multiple relationship types present
    - Verify `CANON/DEPENDENCY_GRAPH.md` exists

    **FOUND-05: Outside resource survey documented with go/no-go decisions**
    - Verify `CANON/RESOURCE_SURVEY.jsonl` exists with valid JSONL
    - Verify records have decision field (adopt/defer/reject)
    - Verify 5 categories covered
    - Verify `CANON/RESOURCE_SURVEY.md` exists
