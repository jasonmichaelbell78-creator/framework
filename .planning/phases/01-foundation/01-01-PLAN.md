<task type="auto">
  <name>Task 2: Create Zod Schemas and framework.config.json</name>
  <files>CANON/schemas/framework-config.schema.ts, CANON/schemas/base-record.schema.ts, framework.config.json</files>
  <action>
    1. Create `CANON/schemas/base-record.schema.ts`:
       - Define universal JSONL base fields extracted from `scripts/config/audit-schema.json`
       - Must include: `id` (string, UUID format), `timestamp` (ISO 8601 string), `version` (semver string)
       - Optional fields: `source` (string), `category` (string), `status` (string)
       - Export the schema as `baseRecordSchema` and the inferred type as `BaseRecord`
       - Use Zod 4 syntax: `import { z } from "zod"`

    2. Create `CANON/schemas/framework-config.schema.ts`:
       - Nested structure per CONTEXT.md decisions:
         ```
         { "project": {}, "systems": { "hooks": {}, "skills": {}, "agents": {}, "scripts": {}, "eslint": {}, "tdms": {} } }
         ```
       - `project.name` is REQUIRED (string, min length 1)
       - `project.repo`, `project.environment`, `project.description` are optional strings
       - Each system section has sensible defaults matching the current codebase layout
       - `systems.hooks.preCommit.preset` should be an enum: "starter" | "standard" | "full"
       - Export the schema as `frameworkConfigSchema` and inferred type as `FrameworkConfig`
       - Export a `validateConfig` function that takes unknown input, parses with the schema, and returns `{ success: boolean, data?: FrameworkConfig, errors?: string[] }`

    3. Create `framework.config.json`:
       - Populate with actual values for this project (name: "framework", repo: "jasonmichaelbell78-creator/framework", etc.)
       - Include all system sections with current directory paths matching the actual codebase layout
       - Do NOT add a `$schema` reference yet (JSON Schema generation deferred per research recommendations)

    4. Create a minimal `framework.config.local.json.example` showing the override pattern:
       ```json
       {
         "systems": {
           "hooks": {
             "preCommit": { "preset": "starter" }
           }
         }
       }
       ```
       This serves as documentation for the local override pattern.

  </action>
  <verify>

    - Both schema files export named schemas and types
    - framework.config.json has project.name set to "framework"
  </verify>
